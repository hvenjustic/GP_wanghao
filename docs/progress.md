## 1. 阶段判断

- `P0`：已基本完成。真实登录权限、订单数据库读写、状态流转、审计日志、低代码治理基础和规则设计工作台都已经落地。
- `P1`：已部分完成。低代码配置 CRUD 与版本治理、规则设计器与试运行已完成，但运行时、差异对比、复杂规则执行和订单效率能力仍未补齐。
- `P2`：基本未开始。第三方同步、物流回调、售后、多租户、运营统计看板等仍主要停留在规划层。
- 当前仓库已不是纯文档或演示项目，而是一个可运行的业务后台原型，默认数据源为 Prisma + PostgreSQL。

## 2. 已完成部分

### 2.1 工程与基础设施

- `已完成`：初始化 Next.js App Router + TypeScript 单仓项目。
- `已完成`：完成 `pnpm`、TypeScript、ESLint、Next.js 基础配置。
- `已完成`：建立 `src/app`、`src/components`、`src/features`、`src/lib`、`src/server`、`prisma` 目录结构。
- `已完成`：补充 `.env.example`、`.env.local`、`.env` 运行配置，默认使用远程 PostgreSQL。
- `已完成`：完成 Prisma Client、`db:push`、`db:seed` 和基础健康检查接口 `/api/health`。

### 2.2 页面与交互骨架

- `已完成`：实现项目总览页 `/`、登录页 `/login`、无权限页 `/forbidden`。
- `已完成`：实现订单列表页 `/orders` 与详情页 `/orders/[id]`。
- `已完成`：实现用户权限页 `/users`。
- `已完成`：实现低代码配置页 `/meta`。
- `已完成`：实现规则工作台 `/rules`。
- `已完成`：实现审计日志页 `/audit-logs` 与规则日志页 `/rule-logs`。

### 2.3 登录与权限

- `已完成`：实现基于数据库用户、角色、权限的真实登录校验。
- `已完成`：实现登录、退出、中间件登录拦截和页面级权限守卫。
- `已完成`：实现菜单权限、页面权限和接口鉴权。
- `已完成`：实现用户状态启停、角色调整、密码重置基础版。
- `已完成`：初始化管理员、运营、审核、配置四类账号、角色与权限关系。

### 2.4 订单业务闭环

- `已完成`：订单列表、详情、状态流转默认走 Prisma 读写。
- `已完成`：支持审核通过、审核驳回、锁单、解锁、异常标记、解除异常、分仓、发货等核心动作。
- `已完成`：支持批量审核通过、批量锁单、批量解锁、批量发货基础版。
- `已完成`：支持订单列表按当前筛选条件导出 CSV。
- `已完成`：支持订单导入模板下载、CSV 上传入口和基础导入校验结果回显。
- `已完成`：支持批量动作逐条结果回显，可在订单列表页查看成功/失败明细。
- `已完成`：支持关键动作原因校验，收紧驳回、锁单、解锁、改仓等操作约束。
- `已完成`：补充首轮订单业务约束收口，异常单不可分仓/发货，未分仓订单不可直接发货。
- `已完成`：补充异常处理联调，列表页与详情页统一展示异常原因、最近处理记录和恢复建议。
- `已完成`：操作日志、审计日志、物流记录和规则命中结果可落库查询。

### 2.5 低代码配置平台

- `已完成`：实体、字段、页面配置 CRUD。
- `已完成`：页面级版本治理，支持预览、发布、克隆新版本、回滚。
- `已完成`：实体/字段级版本治理，支持快照历史、依赖分析和按快照回滚。
- `已完成`：元数据依赖检测、快照落库和审计留痕。
- `已完成`：订单详情页低代码运行时基础版，已能按已发布实体/字段/页面配置动态渲染扩展字段区块。
- `已完成`：低代码运行时只加载已发布版本，草稿字段和草稿页面不会直接影响订单详情页。
- `已完成`：订单列表页已接入发布版低代码运行时，主表可按 `order_extension_list` 的已发布配置加载扩展列。
- `已完成`：低代码页面生效链路基础版已打通，订单列表页与详情页都会优先加载已发布页面版本，自动忽略未发布字段。
- `已完成`：低代码配置差异对比，实体、字段和页面版本现在都可以直接查看当前配置与历史快照/历史版本之间的差异项。
- `已完成`：低代码跨对象依赖拓扑，当前已能查看实体下字段、页面版本、规则版本之间的引用关系，并把规则依赖纳入字段删除约束。
- `已完成`：低代码批量发布与批量回滚，`/meta` 已支持按对象集合批量治理，并返回逐项执行结果。

### 2.6 规则编排引擎

- `已完成`：规则定义管理、版本保存、克隆、发布、回滚和试运行。
- `已完成`：React Flow 设计器，支持节点、连线和图结构存储。
- `已完成`：节点配置语义、字段/操作符/动作模板和模板载入能力。
- `已完成`：条件表达式库，条件节点已支持 `comparison / group / not` 三类表达式，并接入保存校验、试运行求值和订单规则执行链路。
- `已完成`：多分支执行器，分支节点已支持 `branches + defaultTarget` 路由配置，并接入规则保存校验、试运行和订单动作链路。
- `已完成`：订单规则执行器，已接入审核通过、发货前校验、解除锁单、解除异常等订单动作链路。
- `已完成`：示例发布规则 `RULE_SHIPMENT_EXPRESS_GUARD`，可演示加急订单的发货前顺丰校验、分支放行与自动锁单。

## 3. 进行中部分

- `进行中`：订单模块已完成核心闭环，并补了列表导出、导入模板、批量结果回显和异常联调，但导入入库和更多业务约束还未补齐。

## 5. 下一阶段计划

建议下一阶段按以下顺序推进：

1. 先补低代码运行时。
   目标是把当前已落地的订单列表页/详情页运行时继续扩展到表单页，并把发布版配置真正接到表单交互。
2. 然后补规则执行深化能力。
   目标是补规则解释结果和节点逐跳说明，让规则编排从“可配置”走向“可稳定扩展”。
3. 并行补订单剩余效率能力。
   目标是继续收紧业务约束，并评估导入正式入库、去重策略和失败结果回写方案。
4. 最后补用户治理增强和运营支撑。
   目标是完善个人密码维护、角色治理、仓配工作台和统计看板。
